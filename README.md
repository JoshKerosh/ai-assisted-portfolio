# AI-Assisted Portfolio

A modern bilingual professional portfolio built to showcase full-stack development, data engineering, and AI-assisted software engineering workflows through a highly documented and traceable development process.

This project is not only a portfolio website, but also a professional engineering case study demonstrating:

- AI-assisted development workflows
- Spec-Driven Development (SDD)
- OpenSpec methodology
- Scrum-based planning
- Technical documentation practices
- Architecture Decision Records (ADRs)
- CI/CD workflows
- GitHub Projects integration
- Modern frontend engineering

---

# Vision

The goal of this project is to build a highly professional and visually engaging portfolio experience that showcases both technical skills and engineering process maturity.

The portfolio itself will also be treated as a featured project, documenting how it was designed, specified, developed, tested, and deployed using modern AI-assisted engineering workflows.

---

# Positioning

> Full-Stack & Data Developer building modern, reliable, and AI-assisted digital products.

# Project Structure

```txt
app/                          → Next.js App Router
content/                      → Static portfolio content
  case-studies/               → Engineering case study entries
  experience/                 → Work experience data
  profile/                    → Personal profile data
  projects/                   → Project showcase entries
  skills/                     → Skills data
docs/                         → Project documentation
  ai/                         → AI workflow docs and skills
  architecture/               → Architecture diagrams and notes
  decisions/                  → Architecture Decision Records (ADRs)
  deployment/                 → Deployment guides
  design/                     → Design documentation
  process/                    → Engineering process docs
  product/                    → Product planning docs
  scrum/                      → Scrum artifacts
  testing/                    → Testing documentation
openspec/                     → Specifications and changes
  changes/                    → Change specs (OpenSpec format)
public/                       → Static assets
.github/                      → GitHub Actions workflows
```
